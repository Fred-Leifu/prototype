# 260601商务中心规划-账单

# 需求目标

## 需求背景    

| **需求来源：** | 商务中心 |
| --- | --- |
| **需求描述：** | 现在对账效率低，风险管控薄弱，回款周期长，需要增强账单管理规范，提高工作效率和回款效率，减少和用户在对账环节的沟通成本，降低应收。 |
| **上线预期：** | 保障风险可控：<br>*   将账单设为必要环节<br>    <br>*   订单准确（价格、明细）<br>    <br>*   账单和客户达成一致（合同约定）<br>    <br>*   账单凭证留痕（真实准确）<br>    <br>提高回款效率：<br>*   客户可随时查看所有订单、账单<br>    <br>*   快速完成出账单并完成客户确认<br>    <br>*   账单与订单(报价)一致，发票与账单一致<br>    <br>*   （170+）渠道平台订单与账单关联   （正在和业务确认都有哪些平台可以对接，需要生成账单）<br>    <br>提高工作效率：<br>*   业务流程自动化、对账工作任务化<br>    <br>    *   账单自动生成、自动派发<br>        <br>    *   对账周期策略管理<br>        <br>*   账单风险自动提醒 |

## 需求目标

| **项目目标** | **成功标尺（**以什么条件作为衡量目标达成**）** | **成本预估** |  |
| --- | --- | --- | --- |
| 将账单设为必要环节 | 正常核心结束的订单到开票，全部归集到账单，基于账单完成开票； |  | 账单是内部审计的基础支撑。最高人民法院相关司法解释明确指出，如果买方不认可收货，卖方仅凭发票无法证明已履行交货义务。没有前期账单（对账）确认，企业将面临极大败诉风险。 |
| 订单准确（价格、明细） | 账单金额与订单实际结算金额误差率低于X%。 |  | 确保账单金额与实际交付服务/产品一致，避免因价格争议导致回款延迟 |
| 账单凭证留痕（真实准确） | 所有账单状态变更均有操作日志及凭证可追溯。 |  | 账单作为重要的账款凭证，所有操作（如作废、调整）必须留痕，关键操作需上传凭证，保证合法合规有据可查 |
| 客户可随时查看所有订单、账单 | 客户门户/邮件链接可实时查询历史账单及订单详情。 |  | 先货后款模式下，便捷的查看入口有利于加快账单确认和回款周期，降低公司应收账款风险。 |
| 快速完成出账单并完成客户确认 | 账单生成后X个工作日内完成客户确认率不低于XX%。 |  | 缩短对账周期，提升财务结算效率，加速资金回笼。 |
| 业务流程自动化、对账工作任务化 | 核心系统结束的订单，账单自动生成率100%，超时未确认自动触发提醒。 |  | 减少人工干预，降低出错率，通过系统自动派发任务提醒相关人员处理。 |
| 账单风险自动提醒 | 超期未回款提醒及时率100%。 |  | 针对超账期、超时未确认等异常情况及时预警，防范坏账。 |

## 名词定义

| 名词 | 定义 | 备注 |
| --- | --- | --- |
| 账单 | 未出账账单和对账单的统称 | 客户在公司的欠账。 |
| 未出账账单 | 订单创建->生产->结单期间， | 暂时还不需要付款的订单集合 |
| 对账单 | 需要客户确认并在期限内完成付款的部分，订单结束后的流程 | 期限内客户需要付款的订单集合<br>公司计算应收账款会订单结束到回款核销维度 |
| 报账周期 | 和客户对账的频率，和客户协商确定的每隔一段时间客户确认核对所有结束订单 | 也有一种说法叫 结算周期 |
| 客户账期 | 生成对账单到回款核销完成的时间 | 客户需要在账期内完成对账单确认和回款 |
| 公司账期 | 订单结单到回款所用的周期 | 公司按要求完成订单到收回欠款的时间 |
| 预付款 | 客户在公司的余额 | 可以抵扣未来完成订单应付的钱 |
